PnP(Perspective-n-Point)是求解3D到2D点对运动方法，它描述了当知道$N$个3D空间点及其投影位置时，如何估计相机位姿。而P3P问题，即给出空间中3个点的位置及其投影位置，求解相机位姿。一个示意图如下图所示：

P3P的求解是从余弦定理开始的，设相机坐标中心为点$P$，$A$、$B$和$C$是不共线的3个3D点，$D$为验证3D点，根据余弦定理有如下公式： $$ \begin{array}{l}P A^{2}+P B^{2}-2 \cdot P A \cdot P B \cdot \cos <a, b>=A B^{2} \\P A^{2}+P C^{2}-2 \cdot P A \cdot P C \cdot \cos <a, c>=A C^{2} \\P B^{2}+P C^{2}-2 \cdot P B \cdot P C \cdot \cos <b, c>=B C^{2}\end{array} $$ 对上式同除$PC^2$，并且设$y=\frac {PB}{PC}$，$x=\frac {PA}{PC}$。

则有： $$ \begin{array}{l}x^{2}+y^{2}-2 \cdot x \cdot y \cdot \cos <a, b>=\frac{A B^{2}}{P C^{2}} \\x^{2}+1-2 \cdot x \cdot \cos <a, c>=\frac{A C^{2}}{P C^{2}} \\y^{2}+1-2 \cdot y \cdot \cos <b, c>=\frac{B C^{2}}{P C^{2}}\end{array} $$ 设$u$、$v$和$w$分别为如下，并进行替换。 $$ u=\frac{A B^{2}}{P C^{2}}, v=\frac{B C^{2}}{A B^{2}}, w=\frac{A C^{2}}{A B^{2}} $$ 得： $$ \begin{array}{l}x^{2}+y^{2}-2 \cdot x \cdot y \cdot \cos <a, b>=u \\x^{2}+1-2 \cdot x \cdot \cos <a, c>=w u \\y^{2}+1-2 \cdot y \cdot \cos <b, c>=v u\end{array} $$ 将上式中第一个式子带入第2和第3个式子，得： $$ \begin{array}{l}(1-w) x^{2}-w \cdot y^{2}-2 \cdot x \cdot \cos <a, c>+2 \cdot w \cdot x \cdot y \cdot \cos <a, b>+1=0 \\(1-v) y^{2}-v \cdot x^{2}-2 \cdot y \cdot \cos <b, c>+2 \cdot v \cdot x \cdot y \cdot \cos <a, b>+1=0\end{array} $$ 在P3P问题中，3D到2D点的坐标是已知的，因此，$w, v, \cos <a, c>, \cos <b, c>, \cos <a, b>$都是已知量。$AB$，$BC$，$AC$可以通过输入的3D点进行求解，输入的2D点可以求解三个余弦值。

首先根据2D坐标求解余弦值，将像素坐标变换到归一化坐标。 $$ a_{x}=\frac{a_{u}-c_{x}}{f_{x}} \\a_{y}=\frac{a_{v}-c_{y}}{f_{y}} \\a_{z}=1 $$ 然后进行$L2$归一化过程(即数学上的归一化)： $$ N_{a}=\sqrt{\left(a_{x}^{2}+a_{y}^{2}+a_{z}^{2}\right)} \\a_{x}^{\prime}=\frac{a_{x}}{N_{a}} \quad a_{y}^{\prime}=\frac{a_{y}}{N_{a}} \quad a_{z}^{\prime}=\frac{a_{z}}{N_{a}} $$ 因此可以获得余弦值： $$ \cos <\mathrm{a}^{\prime}, \mathrm{b}^{\prime}>=a_{x}^{\prime} \cdot b_{x}^{\prime}+a_{y}^{\prime} \cdot b_{y}^{\prime}+c_{x}^{\prime} \cdot c_{y}^{\prime} $$ $\cos <\mathrm{b}^{\prime}, \mathrm{c}^{\prime}>\quad \cos <\mathrm{a}^{\prime}, \mathrm{c}^{\prime}>$也是同理可求。

然后根据3D坐标求解$AB$，$AC$和$BC$的值，以$AB$为例： $$ A B=\sqrt{\left(\left(a_{x}-b_{x}\right)^{2}+\left(a_{y}-b_{y}\right)^{2}+\left(a_{z}-b_{z}\right)^{2}\right)} $$

接下来求解二元二次方程，这个方程在数学上是可以求解的，需要用到Wu Ritt零点分解法，其中等效方程为：

$$ {\begin{array}{l}a_{4} x^{4}+a_{3} x^{3}+a_{2} x^{2}+a_{1} x^{1}+a_{0}=0 \b_{1} y-b_{0}=0\end{array} $$

其中$a1$到$a4$都是已知的，因此可以求得x和y的值，4次方程理论上有4组解，但只有一组是合适的。在求得x和y的之后，就可以求取$PA$、$PB$、$PC$的值，根据下面公式，$AB$已知，可以先求出$PC$再求解$PB$和$PA$： $$ x^{2}+y^{2}-2 \cdot x \cdot y \cdot \cos <a, b>=\frac{A B^{2}}{P C^{2}} \quad y=\frac{P B}{P C}, x=\frac{P A}{P C} $$ 我们需要的是$A$、$B$、$C$在相机坐标下的坐标，而不是$PA$、$PB$、$PC$的长度，因此还根据长度求取点的坐标，求解方法是利用向量公式： $$ A=\vec{a} \cdot|P A| $$ 其中a是单位向量，$||PA||$是模值，因此获得$A$在相机坐标下的坐标。

最后求得了A，B，C的坐标就可以通过世界坐标系到当前相机坐标的变换求解相机位姿，注意上面求得了4组解，这里需要使用D点确认哪组解是最合适的。

        M_orig[2][1] = lengths[i][2] * mv2;
        M_orig[2][2] = lengths[i][2] * mk2;

        //计算每个解对应的位姿矩阵R，t
        if (!align(M_orig, X0, Y0, Z0, X1, Y1, Z1, X2, Y2, Z2, R[nb_solutions], t[nb_solutions]))
            continue;

        nb_solutions++;
    }

使用吴消元法求解$PA$、$PB$、$PC$的距离：

/// Given 3D distances between three points and cosines of 3 angles at the apex, calculates
/// the lentghs of the line segments connecting projection center (P) and the three 3D points (A, B, C).
/// Returned distances are for |PA|, |PB|, |PC| respectively.
/// Only the solution to the main branch.
/// Reference : X.S. Gao, X.-R. Hou, J. Tang, H.-F. Chang; "Complete Solution Classification for the Perspective-Three-Point Problem"
/// IEEE Trans. on PAMI, vol. 25, No. 8, August 2003
/// \param lengths3D Lengths of line segments up to four solutions.
/// \param dist3D Distance between 3D points in pairs |BC|, |AC|, |AB|.
/// \param cosines Cosine of the angles /_BPC, /_APC, /_APB.
/// \returns Number of solutions.
/// WARNING: NOT ALL THE DEGENERATE CASES ARE IMPLEMENTED

int p3p::solve_for_lengths(double lengths[4][3], double distances[3], double cosines[3])
{
    //吴消元法，数据准备
    double p = cosines[0] * 2;
    double q = cosines[1] * 2;
    double r = cosines[2] * 2;

    double inv_d22 = 1. / (distances[2] * distances[2]);
    double a = inv_d22 * (distances[0] * distances[0]);
    double b = inv_d22 * (distances[1] * distances[1]);

    double a2 = a * a, b2 = b * b, p2 = p * p, q2 = q * q, r2 = r * r;
    double pr = p * r, pqr = q * pr;

    // Check reality condition (the four points should not be coplanar)
    if (p2 + q2 + r2 - pqr - 1 == 0)
        return 0;

    double ab = a * b, a_2 = 2*a;

    double A = -2 * b + b2 + a2 + 1 + ab*(2 - r2) - a_2;

    //A, B, C, D, E 为四次多项式的系数；
    // Check reality condition
    if (A == 0) return 0;

    double a_4 = 4*a;

    double B = q*(-2*(ab + a2 + 1 - b) + r2*ab + a_4) + pr*(b - b2 + ab);
    double C = q2 + b2*(r2 + p2 - 2) - b*(p2 + pqr) - ab*(r2 + pqr) + (a2 - a_2)*(2 + q2) + 2;
    double D = pr*(ab-b2+b) + q*((p2-2)*b + 2 * (ab - a2) + a_4 - 2);
    double E = 1 + 2*(b - a - ab) + b2 - b*p2 + a2;

    double temp = (p2*(a-1+b) + r2*(a-1-b) + pqr - a*pqr);
    double b0 = b * temp * temp;
    // Check reality condition
    if (b0 == 0)
        return 0;

    //求解四次多项式；
    double real_roots[4];
    int n = solve_deg4(A, B, C, D, E,  real_roots[0], real_roots[1], real_roots[2], real_roots[3]);

    if (n == 0)
        return 0;

    int nb_solutions = 0;
    double r3 = r2*r, pr2 = p*r2, r3q = r3 * q;
    double inv_b0 = 1. / b0;

    // For each solution of x
    for(int i = 0; i < n; i++) {
        double x = real_roots[i];

        // Check reality condition
        if (x <= 0)
            continue;

        double x2 = x*x;
　　　　//对应附录中的b1
        double b1 =
            ((1-a-b)*x2 + (q*a-q)*x + 1 - a + b) *
            (((r3*(a2 + ab*(2 - r2) - a_2 + b2 - 2*b + 1)) * x +

            (r3q*(2*(b-a2) + a_4 + ab*(r2 - 2) - 2) + pr2*(1 + a2 + 2*(ab-a-b) + r2*(b - b2) + b2))) * x2 +

            (r3*(q2*(1-2*a+a2) + r2*(b2-ab) - a_4 + 2*(a2 - b2) + 2) + r*p2*(b2 + 2*(ab - b - a) + 1 + a2) + pr2*q*(a_4 + 2*(b - ab - a2) - 2 - r2*b)) * x +

            2*r3q*(a_2 - b - a2 + ab - 1) + pr2*(q2 - a_4 + 2*(a2 - b2) + r2*b + q2*(a2 - a_2) + 2) +
            p2*(p*(2*(ab - a - b) + a2 + b2 + 1) + 2*q*r*(b + a_2 - a2 - ab - 1)));

        // Check reality condition
        if (b1 <= 0)
            continue;

        double y = inv_b0 * b1;
        double v = x2 + y*y - x*y*r;

        if (v <= 0)
            continue;

        double Z = distances[2] / sqrt(v);
        double X = x * Z;
        double Y = y * Z;

        lengths[nb_solutions][0] = X;
        lengths[nb_solutions][1] = Y;
        lengths[nb_solutions][2] = Z;

        nb_solutions++;
    }

    return nb_solutions;
}

    for(int i = 0; i < n; i++) {
        double X3p = Rs[i][0][0] * X3 + Rs[i][0][1] * Y3 + Rs[i][0][2] * Z3 + ts[i][0];
        double Y3p = Rs[i][1][0] * X3 + Rs[i][1][1] * Y3 + Rs[i][1][2] * Z3 + ts[i][1];
        double Z3p = Rs[i][2][0] * X3 + Rs[i][2][1] * Y3 + Rs[i][2][2] * Z3 + ts[i][2];
        double mu3p = cx + fx * X3p / Z3p;
        double mv3p = cy + fy * Y3p / Z3p;
        //通过R，t计算第4个点的重投影误差选择合理的解
        double reproj = (mu3p - mu3) * (mu3p - mu3) + (mv3p - mv3) * (mv3p - mv3);
        //选择重投影误差最小的解
        if (i == 0 || min_reproj > reproj) {
            ns = i;
            min_reproj = reproj;
        }
    }
